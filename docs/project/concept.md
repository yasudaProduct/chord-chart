# プロジェクトコンセプト

ChordBook のプロジェクト概要とビジョンを説明します。

## コンセプト

> **「作りやすく、演奏しやすく、管理しやすい」自分だけのコード譜ライブラリ**

既存のコード譜サイトへの不満を解消し、カバー曲もオリジナル曲も一元管理できる、広告なしのユーザーフレンドリーなアプリケーション。

## ターゲットユーザー

- コード記法（C、Dm など）を使って演奏するミュージシャン
- 既存のコード譜サイトに不満を持つユーザー
- オリジナル曲とカバー曲のアレンジを一元管理したい人

## 課題（ペインポイント）

### 既存曲を演奏する人の悩み

| 課題 | 詳細 |
|------|------|
| 広告が多い | 既存サイトは広告が多く、閲覧の邪魔になる |
| 複数サイトの使い分け | 曲によって複数のコード譜サイトを行き来する必要がある |
| スマホで見づらい | スタジオでスマホを見ながら演奏する際、表示が見づらい |
| 印刷品質が悪い | 印刷すると見切れたり、レイアウトが崩れたりする |
| カスタマイズできない | 公開されているアレンジを自分好みに編集できない |

### カバー曲とオリジナル曲の両方を持つ人の悩み

| 課題 | 詳細 |
|------|------|
| 二重管理 | サイトから取得したコード譜と自作のコード譜を別々に管理している |

## 提供価値

4つの柱でユーザーの課題を解決します。

### 1. 作る（Create）

- **コード補完**: コード入力時に候補を自動表示
- **スマート予測**: 進行パターンから次のコードを予測
- **直感的なエディター**: シンプルで使いやすい編集画面

### 2. 演奏する（Perform）

- **クリーンなデザイン**: 広告なしで見やすい表示
- **オートスクロール**: BPM に合わせた自動スクロール
- **演奏モード**: 演奏に最適化された表示

### 3. 管理する（Manage）

- **一元管理**: カバー曲もオリジナル曲も1箇所で管理
- **共有機能**: バンドメンバーとコード譜を共有
- **検索・フィルター**: 曲をすぐに見つけられる

### 4. 出力する（Output）

- **高品質印刷**: 見切れのない綺麗な印刷
- **PDF出力**: 印刷用にPDFをエクスポート

## 開発ロードマップ

### Phase 1: MVP（最小限の実用製品）

| カテゴリ | 機能 |
|----------|------|
| エディター | 歌詞+コード形式、小節形式 |
| 構成管理 | セクションマーカー（イントロ、Aメロ等） |
| 入力支援 | コード補完、スマート予測 |
| 管理機能 | 曲の保存・編集・削除 |
| 出力機能 | 印刷対応 |

### Phase 2: 演奏支援＆共有

| カテゴリ | 機能 |
|----------|------|
| 演奏支援 | オートスクロール、BPM同期 |
| 共有機能 | 公開設定（非公開/URL共有/公開） |
| インポート | 外部フォーマットからの取り込み |

### Phase 3: コミュニティ＆拡張

| カテゴリ | 機能 |
|----------|------|
| コミュニティ | 公開曲の検索、ユーザーフォロー |
| 高度な機能 | 音声解析からのコード検出、移調 |
| モバイル | iOS/Android アプリ |

## ビジネスモデル（検討中）

| モデル | 説明 |
|--------|------|
| フリーミアム | 基本機能は無料、高度な機能は有料 |
| サブスクリプション | 月額/年額課金 |
| 買い切りアプリ | モバイルアプリは買い切り |

## 競合との差別化

| 観点 | 既存サービス | ChordBook |
|------|-------------|-----------|
| 広告 | あり（多い） | なし |
| オリジナル曲 | 非対応 | 対応 |
| カスタマイズ | 不可 | 自由に編集可能 |
| 印刷品質 | 低い | 高品質 |
| オフライン | 非対応 | 対応予定 |
